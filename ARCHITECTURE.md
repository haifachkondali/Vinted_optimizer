# Architecture - Vinted Optimizer\n\n## Overview\n\nVinted Optimizer is a modular pipeline for analyzing and optimizing Vinted listings with AI-driven recommendations.\n\n## Architecture Diagram\n\n```\nInput Sources\n├── --user (scraping)\n├── --file (JSON)\n├── --export-folder (GDPR)\n└── --export-zip (GDPR)\n    |\n    v\n[Data Loader]\n├── scraper.py      (Vinted web scraping)\n├── html_parser.py  (GDPR export parsing)\n└── models.py       (ItemInput dataclass)\n    |\n    v\n[Analysis Pipeline]\n├── utils.py        (Text cleaning, keyword extraction, brand inference)\n├── analyser.py     (Core scoring engine)\n│   ├── attractiveness_score()\n│   ├── photo_analysis()\n│   ├── pricing_strategy()\n│   ├── build_titles()\n│   ├── rewrite_description()\n│   ├── marketing_strategy()\n│   └── compute_global_score()\n└── constants.py    (Brand lists, category maps, stopwords)\n    |\n    v\n[Output Generation]\n├── reporter.py     (Markdown + JSON report generation)\n└── models.py       (AnalysisOutput dataclass)\n    |\n    v\nOutput Files\n├── analyse_NNN.md           (Detailed markdown reports)\n└── analyses_completes.json  (Consolidated JSON)\n```\n\n## Module Breakdown\n\n### core/models.py\n**Purpose**: Data models\n\n- `ItemInput`: Input article data (titre, description, prix, images, etc.)\n- `AnalysisOutput`: Analysis results (recommendations, scores, reports)\n\n### core/scraper.py\n**Purpose**: Web scraping integration\n\n- `scrape_user_items()`: Scrapes public Vinted profiles using vinted_scraper library\n- Handles errors gracefully, retries with delay\n- Returns list of ItemInput objects\n\n### core/html_parser.py\n**Purpose**: GDPR export parsing\n\n- `parse_items_html()`: Extracts items from HTML tables\n- `load_from_export_folder()`: Processes unzipped GDPR exports\n- `load_from_zip()`: Decompresses and processes ZIP files\n- Smart HTML file detection for various export versions\n\n### core/utils.py\n**Purpose**: Text processing and inference\n\n- `clean_text()`: Normalize whitespace\n- `extract_keywords()`: Extract unique keywords from text\n- `infer_brand()`: Detect brand from title or brand field\n- `infer_category()`: Infer category from text\n- `demand_level()`: Determine market demand (high/mid/low)\n\n### core/constants.py\n**Purpose**: Reference data\n\n- `HIGH_DEMAND`: Premium brands (Nike, Zara, etc.)\n- `MID_DEMAND`: Popular mass-market brands (H&M, Bershka, etc.)\n- `STATE_BOOST`: Condition scoring modifiers\n- `CATEGORY_MAP`: Category inference mappings\n- `STOPWORDS`: French stopwords for keyword extraction\n\n### core/analyser.py\n**Purpose**: Core analysis engine\n\n**Scoring Components**:\n- `attractiveness_score()`: 0-10 based on brand, price, condition, description length, photos\n- `photo_analysis()`: Evaluates photo count and provides improvement suggestions\n- `pricing_strategy()`: Recommends optimal prices based on demand level\n- `build_titles()`: Generates 3 SEO-optimized title alternatives\n- `rewrite_description()`: Creates marketing-focused description template\n- `marketing_strategy()`: Suggests hashtags, timing, bundling strategies\n- `compute_global_score()`: Weighted average of all components\n\n**Integration**:\n- `analyse_item()`: Main pipeline combining all analysis functions\n- Returns AnalysisOutput with complete recommendations\n\n### core/reporter.py\n**Purpose**: Report generation\n\n- `format_report()`: Converts analysis to formatted markdown\n- `save_reports()`: Saves individual reports and consolidated JSON\n\n### main.py\n**Purpose**: CLI entry point and orchestration\n\n- `run()`: Typer CLI command\n- Handles input source selection\n- Manages progress bars\n- Coordinates loading, analysis, and output\n- Error handling and user feedback\n\n## Data Flow\n\n1. **Input** → User provides data source (user/file/folder/zip)\n2. **Loading** → Appropriate loader extracts ItemInput list\n3. **Analysis** → analyser.py processes each item with 5 scoring components\n4. **Scoring** → Results combined into AnalysisOutput\n5. **Reporting** → Reports formatted and saved to output directory\n\n## Scoring System\n\n### Component Scores (0-10)\n\n- **Attractiveness**: Brand popularity, price tier, condition, description length, photo presence\n- **Photo Quality**: Based on count; 0 photos = 3.5/10, each photo adds 0.8/10\n- **SEO Score**: Title length >= 10 chars = 7.5/10, else 5.5/10\n- **Pricing**: Price renseigned = 7.0/10, else 5.5/10\n- **Conversion Potential**: Average of all components\n\n### Global Score\n\n```\nGlobal = (Attractiveness + Photo + SEO + Pricing + Conversion) / 5\n```\n\n### Interpretation\n\n- **7.5-10**: Strong sales potential\n- **6.0-7.5**: Good potential, optimize presentation\n- **< 6.0**: Needs significant work\n\n## Extension Points\n\nFuture enhancements could be added to:\n\n1. **New Data Sources**: Add loaders in scraper.py or html_parser.py\n2. **Scoring Algorithms**: Enhance analyser.py scoring functions\n3. **Brand/Category Data**: Extend constants.py lists\n4. **Report Formats**: Add new formatters in reporter.py (PDF, HTML, etc.)\n5. **AI Integration**: Add ML models for price prediction, demand forecasting\n\n## Performance Considerations\n\n- Scraping: ~500ms per item (with 1.5s delay)\n- Analysis: ~50ms per item\n- Report generation: ~20ms per report\n- Memory: ~100KB per item in memory\n\nFor 100 items:\n- Scraping: ~2.5 minutes\n- Analysis: ~5 seconds\n- Total: ~2.5 minutes wall-clock time\n
